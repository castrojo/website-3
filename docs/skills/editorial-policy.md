# Editorial Policy

The user authors all creative and editorial content.

## Core Process

1. Find the authoritative source: direct user text, existing repository content, or cited upstream material.
2. Preserve exact wording, attribution, order, and provenance.
3. Organize supplied material without inventing connective prose, headings, chapter names, summaries, or tone.
4. Use standard lorem ipsum only when the user explicitly requests a placeholder.
5. If authored material appears missing, recover it from the working tree, Git history or reflog, then editor history or backups. Ask the user only after recovery fails.

## Prohibited

- Fiction, lore, dialogue, character details, or chapter names generated by an agent.
- Invented quotes, citations, attributions, or technical claims.
- Marketing or release-note narrative not supplied by the user.
- Machine-generated translations presented as authored copy.
- Rewriting recovered prose.

## Verification

- [ ] Every creative or editorial string matches an authoritative source.
- [ ] Quotes and attributions remain exact.
- [ ] No invented headings, summaries, or connective prose were added.
- [ ] Missing authored work was recovered before asking for re-entry.

## Sources

- Direct user input
- Existing repository content and Git history
