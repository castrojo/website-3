# projectbluefin/website Agent Contract

This repository builds the public Project Bluefin website and its unlisted sub-apps with Vue, TypeScript, Vite, Tailwind, and SCSS.

## Start Here

1. Read source before describing repository behavior.
2. Open `docs/SKILL.md`.
3. Load only the skill matching the task.
4. Stop at any human or design gate before editing.
5. Load `docs/skills/build-verify-deploy.md` before reporting completion.

For external tools, libraries, and platform APIs, read current authoritative documentation through Context7 before implementation.

## Universal Rules

- No emojis in code, configuration, documentation, commits, pull requests, or agent responses.
- Never invent fiction, lore, quotes, attributions, marketing prose, or release-note narrative. The user authors editorial content.
- A content, data, behavior, or bug-fix request does not authorize a redesign.
- Wolves design is frozen. Read `docs/skills/wolves-content-maintenance.md` before any Wolves change.
- Established-page visual changes require explicit human approval before editing.
- Explicitly requested new pages or components may use the design workflow in `docs/skills/design-authority.md`.
- Never modify, stage, restore, or commit unrelated dirty worktree files.
- Public runtime asset requests use `import.meta.env.BASE_URL`, not absolute `/` paths.
- `src/components/wolves/wallpapers-list.ts` is generated by `scripts/generate-wallpapers.js`; never edit it directly.

## Verified Commands

| Purpose | Command |
|---|---|
| Install | `npm install --include=dev` |
| Develop | `npm run dev` |
| Format and lint | `npm run lint:fix` |
| Type-check | `npm run typecheck` |
| Test | `npm run test:run` |
| Build | `npm run build` |
| Preview | `npm run preview` |

Use npm for repository work. `npm run test:run` may rewrite `public/dakota-versions.json`; leave unrelated changes unstaged.

- Routine content, assets, data, downloads, and unlisted sub-apps: `docs/skills/site-maintenance.md`
- Visual and component work: `docs/skills/design-authority.md`
- Wolves content: `docs/skills/wolves-content-maintenance.md`

- Skill maintenance: `docs/skills/skill-improvement.md`

Dakota, Bluespeed, and Knuckle are intentionally unlisted or `noindex`. Do not promote them through navigation or sitemaps without explicit approval.

## Git and Deployment

Push directly to `upstream main` (`projectbluefin/website`) by default unless the user explicitly requests a branch.

Use Conventional Commits. Every AI-authored commit includes:

```text
Assisted-by: GPT-5.6 Sol via GitHub Copilot CLI
Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>
```

Stage explicit paths only. Never use `git add .` or `git add -A`.

A local build or successful push is not production completion. Verify the exact pushed SHA's `Deploy to GitHub Pages` workflow through `docs/skills/build-verify-deploy.md`.

## Self-Improvement

Reusable discoveries update the closest skill in the same change. Do not create changelogs, session logs, plan files, or append-only agent notes in the repository.
